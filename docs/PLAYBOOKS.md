# Playbooks

> 中文版: [PLAYBOOKS.zh-CN.md](PLAYBOOKS.zh-CN.md)

A playbook is a **curated bundle of skills + a suggested order**. Use it when you want a repeatable workflow (not one-off prompts).

Best for: PMs running a full product cycle (discovery → decision → spec → execution → launch → learning).

## How to use a playbook

1) Install skills (recommended: `skills-all.zip` from GitHub Releases).
2) Pick one playbook and treat each skill’s deliverables as inputs to the next skill.
3) Save outputs as files in your repo (PRD, decision memo, plan, checklist) so you can iterate and review.
