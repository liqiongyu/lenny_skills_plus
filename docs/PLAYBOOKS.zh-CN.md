# Playbooks（推荐组合）

> English version: [PLAYBOOKS.md](PLAYBOOKS.md)

Playbook 是 **一组精选 skills + 建议使用顺序**。适用于你想把 skills 当作一个可复用的工作流来用（而不是临时问一个问题）。

适用：PM 跑完整产品周期（调研 → 决策 → 写规格 → 推进交付 → 发布 → 复盘学习）。

**Starter kit（10 个 skills）**

- [analyzing-user-feedback](../skills/analyzing-user-feedback/)
- [defining-product-vision](../skills/defining-product-vision/)
- [positioning-messaging](../skills/positioning-messaging/)
- [prioritizing-roadmap](../skills/prioritizing-roadmap/)

**建议顺序**
1) 调研： [conducting-user-interviews](../skills/conducting-user-interviews/)、[analyzing-user-feedback](../skills/analyzing-user-feedback/)
2) 问题定义： [problem-definition](../skills/problem-definition/)
3) 策略与叙事： [defining-product-vision](../skills/defining-product-vision/)、[positioning-messaging](../skills/positioning-messaging/)
4) 优先级与决策： [prioritizing-roadmap](../skills/prioritizing-roadmap/)
5) 写规格与对齐： [writing-prds](../skills/writing-prds/)、[cross-functional-collaboration](../skills/cross-functional-collaboration/)
6) 推进交付与发布： [managing-timelines](../skills/managing-timelines/)、[shipping-products](../skills/shipping-products/)

## How to use a playbook

1) 安装 skills（推荐：从 GitHub Releases 下载 `skills-all.zip`）。
2) 选择一个 playbook，把每个 skill 的交付物当作下一个 skill 的输入。
3) 把输出保存为你项目里的文件（PRD、决策 memo、计划、清单），便于迭代与评审。
